# Отчет по лабораторной работе №6 МобПри

## Выполнили: Челяпов и Кузнецов

[Скрины для отчета](pfotootchetlr6.docx)

## Тема: 
Первый экран на Compose (Text, Button, Column, Row, Box), Previews, модификаторы для стилизации, композиция UI

## Цель работы
Освоить базовые элементы Jetpack Compose, научиться создавать адаптивные интерфейсы, использовать модификаторы, превью и компонентный подход.

## Выполненные задания
### Часть 1. Базовые элементы
1.1 Первый экран: создана функция GreetingScreen() с Column, содержащим Text и Button. Размещение вертикальное, выравнивание по центру.

1.2 Column, Row, Box:

Добавлены несколько Text и кнопок в Column с отступами spacedBy(8.dp).

Реализована StatsRow() – горизонтальный ряд из трёх текстовых меток с SpaceEvenly.

Создана OverlayBadge() – Box с центральным текстом и бейджем «NEW» в правом верхнем углу.

1.3 Экран «О приложении»: AboutScreen() содержит заголовок, описание (несколько строк) и две кнопки («Версия», «Лицензия») в Row.

### Часть 2. Previews
2.1 Базовый Preview: написаны отдельные функции с @Preview для AboutScreen в светлой и тёмной темах (с помощью uiMode = UI_MODE_NIGHT_YES/NO).

2.2 Preview на разных устройствах: созданы две превью-функции с параметрами device = Devices.PIXEL_4 и device = Devices.NEXUS_9 для демонстрации адаптивности.

### Часть 3. Модификаторы
3.1 Размеры, отступы: StyledCard() – карточка с заголовком, подзаголовком и кнопкой. У Column задана цепочка модификаторов: .fillMaxWidth().padding(16.dp).

3.2 Фон, скругление, тень: добавлены .background(Color.LightGray, RoundedCornerShape(12.dp)), .shadow(4.dp). Порядок модификаторов соблюдён.

3.3 Кликабельность и доступность: к карточке добавлен .clickable { Log.d(...); Toast(...) } и .semantics { contentDescription = ... }.
Проблема: LocalContext.current нельзя использовать внутри лямбды clickable. Решение – вынести получение контекста в тело @Composable функции до модификатора.

### Часть 4. Композиция UI
Создан экран «Профиль пользователя»:

ProfileHeader(name, role) – имя и роль.

ProfileStat(label, value) – строка «метка: значение».

ProfileScreen() – собирает компоненты, добавлена кнопка «Редактировать». Для возможности прокрутки использован Modifier.verticalScroll(rememberScrollState()).

### Часть 5. Итоговое задание – Карточка рецепта
Реализована RecipeCard() с требованиями:

Использованы Column, Row, Box, Text, Button.

Применены модификаторы: fillMaxWidth, padding, background, clip, clickable, shadow, semantics (более пяти).

Два вспомогательных компонента: RecipeHeader(title, description) и RecipeMeta(time, servings).

Две превью-функции (светлая и тёмная тема).

## Выводы
В ходе лабораторной работы приобретены практические навыки разработки UI на Jetpack Compose:

Освоены базовые компоновщики (Column, Row, Box) и их параметры выравнивания.

Изучены модификаторы для стилизации, отступов, фона, кликов и доступности.

Научились быстро отлаживать интерфейс через @Preview без запуска эмулятора.

Закреплён принцип композиции – разбиение экрана на маленькие переиспользуемые @Composable функции.

Получен опыт решения типичных проблем, связанных с контекстом и областью видимости Compose.

Результат: все функциональные требования выполнены, приложение успешно запускается на эмуляторе, превью отображаются корректно.
